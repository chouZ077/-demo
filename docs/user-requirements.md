# 用户需求文档

## 1. 项目概述

本项目是一个面向大学生的高校周边餐饮地图 Web App。

它的核心目标不是做一个泛化的大众餐饮平台，而是围绕“大学生在学校附近吃什么、值不值得吃、怎么快速找到适合自己的店”这个问题，提供一个更直观的地图式浏览体验。

用户进入产品后，首先看到的是一个总地图，地图上展示若干所高校的位置。点击某所高校后，地图会放大到该校周边，并显示附近餐饮店的分布。继续手动缩放后，用户可以查看店名、店铺类型标识，并进入店铺详情页。

## 2. 目标用户

### 核心用户

- 在校大学生
- 刚到新校区、对周边餐饮不熟悉的学生
- 希望快速找到高性价比、高评分餐厅的学生
- 喜欢参考同龄人意见的学生用户

### 次级用户

- 高校周边生活服务内容的整理者
- 刚入学的新生
- 到学校办事、探访或短暂停留的人群

## 3. 用户痛点

1. 外卖平台和点评平台信息很多，但对“离学校近不近、是否适合学生”不够聚焦。
2. 用户通常先知道学校，再想看学校附近有什么店，而不是先按全城搜索。
3. 店铺列表型产品不够直观，难以快速理解高校与餐饮店之间的空间关系。
4. 平台评分能反映总体口碑，但无法体现“大学生是否推荐”这一群体视角。
5. 分类查找时，如果只是平铺列表，用户很难快速切换菜系并比较。

## 4. 核心需求

### 4.1 地图浏览需求

用户希望先看到一个高校总览地图，并能直观知道有哪些学校可选。

用户点击学校后，希望地图自动聚焦到该学校附近，同时显示周边餐饮店位置。

用户继续放大地图后，希望看到更细的店铺信息，包括：

- 店名
- 店铺图标或 logo
- 店铺类别提示，例如水果店、奶茶店、饭店等

### 4.2 店铺详情需求

用户点击店铺后，希望进入一个店铺详情页，并快速了解这家店值不值得去。

详情页至少需要包含：

- 店铺名称
- 店铺类型
- 基本位置说明
- 美团评分
- 大众点评评分
- 1 到 2 条简短点评预览
- 每条点评的星级
- 跳转按钮，引导去对应 App 查看完整评论

点评内容只展示简短截断版本，以省略号结尾，避免页面过长，同时引导用户跳转查看原始内容。

### 4.3 大学生严选需求

用户除了参考平台评分外，还希望知道“大学生到底推不推荐这家店”。

因此详情页需要加入“大学生严选”模块，包含两个可点击选项：

- 推荐
- 不推荐

表现形式为两个不同颜色和表情的卡通星星：

- 笑脸星星表示推荐
- 哭脸星星表示不推荐

每个选项边上显示历史点击数，用户也可以投票。单个用户只允许点击一次，点击后数据会计入统计结果。

### 4.4 分类浏览需求

用户在不通过地图逐个寻找的情况下，也希望按菜系和品类快速筛选餐厅。

因此在初始地图页面两侧需要有一个竖向的分类引导区域，分类项包括但不限于：

- 湘菜
- 川菜
- 粤菜
- 日料
- 韩餐
- 西餐

分类项点击后，打开一个新的悬浮页面，页面内展示该分类下的餐厅列表，并支持下拉浏览。

### 4.5 排序需求

用户在分类页面中，希望对餐厅进行排序比较。

需要支持两种排序方式：

1. 评价数最多
2. 评分最高

评分采用星级形式展示，满分为 5 星。

### 4.6 搜索需求

主页面顶部需要有一个搜索栏，允许用户直接按店名搜索目标餐厅。

### 4.7 右侧悬浮入口需求

主页面最右侧需要有几个圆形悬浮球，用作扩展服务入口，当前包括：

- 美团直播
- 美团酒店
- 美团团购
- 美团优选

这些入口在当前阶段主要承担视觉展示和功能预留作用。

## 5. 主要使用场景

### 场景 A：新生熟悉学校周边

用户刚到学校，不知道周边有哪些吃饭的地方，于是先点开学校地图，再逐步放大查看附近的餐饮店。

### 场景 B：按菜系找店

用户今天只想吃某种菜系，例如川菜，因此直接打开左侧或右侧的分类栏，进入川菜列表页，再按评分最高排序查看。

### 场景 C：判断一家店是否值得尝试

用户点击某家店后，在详情页中综合查看平台评分、简短评论和“大学生严选”投票结果，再决定是否去吃。

### 场景 D：搜索已知店名

用户已经知道某家店的名字，不想通过地图慢慢找，因此直接在顶部搜索栏中输入店名并跳转到结果。

## 6. MVP 范围

第一版建议优先实现以下最小闭环：

1. 高校总地图展示
2. 点击学校后地图聚焦并展示周边餐饮店点位
3. 地图缩放后显示店名和店铺类别图标
4. 店铺详情页
5. 平台评分展示
6. 点评摘要展示
7. 大学生严选推荐/不推荐投票模块
8. 分类悬浮页
9. 分类排序
10. 顶部搜索栏
11. 右侧悬浮球入口展示

## 7. 非目标

当前阶段先不把以下内容作为首要目标：

- 全城市级别的餐饮覆盖
- 完整外卖交易能力
- 用户账户体系的复杂社交功能
- 平台评论全文展示
- 多维度个性化推荐算法

## 8. 产品价值

这个产品的价值主要体现在三个层面：

1. 用高校作为入口，比传统餐饮平台更贴近大学生的真实决策路径。
2. 用地图缩放式浏览，比单纯列表更容易理解餐饮分布。
3. 用“大学生严选”补充平台评分，让产品形成更强的校园社区感。

## 9. 当前假设

由于你目前还没有给出具体高校名单，文档先按“支持多所高校”的方式描述。

后续需要进一步明确：

- 首批上线高校有哪些
- 每所学校覆盖的餐饮范围多大
- 店铺数据来源和评分更新方式
- 点评内容采用手动整理还是接口同步

## 10. 验收标准

满足以下条件，可视为第一阶段产品说明完整：

1. 用户能理解产品首页为什么先展示高校地图。
2. 用户能理解地图缩放后的信息层级变化。
3. 用户能理解店铺详情页需要展示哪些核心信息。
4. 用户能理解“大学生严选”的交互规则和作用。
5. 用户能理解分类、排序、搜索和悬浮入口在页面中的位置与用途。
